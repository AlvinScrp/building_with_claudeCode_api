# 工具使用介绍

## 没有工具的问题

当用户向 Claude 询问当前信息时，它会遇到障碍。例如，如果有人问"加利福尼亚州旧金山的天气怎么样？"Claude 只能回复类似"抱歉，我无法获取最新的天气信息。"这样的内容。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623642%2F06_-_001_-_Introducing_Tool_Use_05.1748623642311.png)当用户需要实时数据时，这会造成令人沮丧的体验——如果 Claude 能够访问当前信息，理论上是可以提供帮助的。

## 工具使用的工作原理

工具使用遵循应用程序与 Claude 之间特定的往返模式。以下是完整的流程：

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623643%2F06_-_001_-_Introducing_Tool_Use_07.1748623643055.png)1. **初始请求：** 您向 Claude 发送问题，并附上如何从外部来源获取额外数据的说明

2. **工具请求：** Claude 分析问题后判断需要更多信息，然后明确说明它需要哪些具体数据
3. **数据检索：** 您的服务器运行代码，从外部 API 或数据库获取所请求的信息
4. **最终响应：** 您将检索到的数据发送回 Claude，Claude 随后结合原始问题和最新数据生成完整的回复

## 天气示例实践

让我们看看这在天气问题中是如何运作的。这个过程会变得更加具体：

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623644%2F06_-_001_-_Introducing_Tool_Use_14.1748623643863.png)当用户询问当前天气时，你在提示词中包含了如何获取天气数据的说明。Claude 识别出它需要当前信息，并请求获取特定位置的天气数据。然后你的服务器调用天气 API 获取实时状况，并将该数据发送回 Claude。最后，Claude 将最新的天气数据与用户的问题结合起来，提供准确、及时的回复。

## 主要优势

* **实时信息：** 访问 Claude 训练期间不可用的当前数据
* **外部系统集成：** 将 Claude 连接到数据库、API 和其他服务
* **动态响应：** 根据最新可用信息提供答案
* **结构化交互：** Claude 清楚地知道自己需要什么信息以及如何询问

工具使用将 Claude 从静态知识库转变为能够处理实时数据的动态助手。这为构建需要当前信息的应用程序开辟了可能性，无论是天气数据、股票价格、数据库查询，还是用户可能需要的任何其他实时信息。

# 项目概述

我们将构建一个实用项目，教会 Claude 如何设置未来日期的提醒。这听起来可能很简单，但它揭示了几个有趣的挑战，我们将使用自定义工具来解决这些问题。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623630%2F06_-_002_-_Project_Overview_00.1748623629884.png)目标很直接：我们希望能够告诉 Claude"为我的医生预约设置一个提醒，时间是下周四"，然后 Claude 回复"好的，我会提醒你。"但要实现这一点，我们需要解决 Claude 在处理时间和提醒方面的一些局限性。

## 为何这具有挑战性

虽然 Claude 知道当前日期，但我们需要解决三个具体问题：

* **时间感知有限：** Claude 可能知道当前日期，但不知道确切时间
* **日期计算问题：** Claude 并不总能很好地处理基于时间的加法运算，尤其是在计算未来很多天的日期时
* **没有提醒功能：** Claude 不知道如何设置提醒——它没有内置的提醒机制

这些限制中的每一个都代表着 Claude 的自然能力与我们提醒系统所需功能之间的差距。工具正是我们弥合这些差距的方式。

## 我们需要的工具

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623630%2F06_-_002_-_Project_Overview_17.1748623630589.png)我们将创建三个独立的工具来处理每个挑战：

* **获取当前日期时间：** Claude 需要精确知道当前的日期和时间
* **为日期时间添加时长：** Claude 在日期时间加法运算上并不完美，因此我们将为其提供一个可靠的工具来完成此任务
* **设置提醒：** 我们需要一种方法来在系统中实际设置提醒

我们将逐一实现这些工具，从最简单的开始。这种方法让我们在构建更复杂的功能之前，先理解工具调用的工作原理。最终，Claude 将能够处理诸如"一周后提醒我"这样的自然语言请求，通过组合这些工具来计算确切时间并设置提醒。

这个项目展示了与 AI 协作的一个关键原则：当模型存在局限性时，我们通过工具来扩展其能力，而不是试图在提示词中绕过这些局限性。

# 工具函数

在使用 Claude 构建 AI 应用时，你通常需要让它能够访问实时信息或执行操作。这就是工具函数的用武之地——它们是 Python 函数，Claude 可以在需要额外数据来帮助用户时调用这些函数。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623640%2F06_-_003_-_Tool_Functions_00.1748623639986.png)上图展示了我们将要实现的三个基本工具：获取当前日期/时间、为日期添加时间间隔，以及设置提醒。让我们从第一个开始。

## 什么是工具函数？

工具函数是一个普通的 Python 函数，当 Claude 判断需要额外信息来帮助用户时，它会自动执行该函数。例如，如果有人问"现在几点了？"，Claude 就会调用你的日期/时间工具来获取当前时间。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623641%2F06_-_003_-_Tool_Functions_06.1748623640850.png)以下是一个天气工具函数的示例。请注意它如何验证输入并提供清晰的错误消息——这些都是重要的最佳实践。

## 工具函数的最佳实践

编写工具函数时，请遵循以下准则：

* **使用描述性名称：** 函数名和参数名都应清晰地表明其用途
* **验证输入：** 检查必需参数是否为空或无效，如果是则抛出错误
* **提供有意义的错误信息：** Claude 能够看到错误信息，并可能使用修正后的参数重新尝试函数调用

验证尤为重要，因为 Claude 会从错误中学习。如果你抛出一个明确的错误，如"Location cannot be empty"，Claude 可能会尝试使用正确的位置值再次调用该函数。

## 构建你的第一个工具函数

让我们创建一个获取当前日期和时间的函数。该函数将接受一个日期格式参数，以便 Claude 可以请求不同格式的时间：

```
def get_current_datetime(date_format="%Y-%m-%d %H:%M:%S"):
    if not date_format:
        raise ValueError("date_format cannot be empty")
    return datetime.now().strftime(date_format)
```

该函数使用 Python 的 datetime 模块获取当前时间，并根据提供的格式字符串进行格式化。默认格式输出为：年-月-日 时:分:秒。

你可以用不同的格式来测试它：

```
# Default format: "2024-01-15 14:30:25"
get_current_datetime()

# Just hour and minute: "14:30"
get_current_datetime("%H:%M")
```

验证检查确保 Claude 不能为日期格式传递空字符串。虽然这种特定错误不太可能发生，但它展示了验证输入并提供有用错误消息的模式，Claude 可以从中学习。

## 下一步

创建函数只是第一步。接下来，你需要编写一个 JSON schema 来向 Claude 描述该函数，然后将其集成到你的聊天系统中。这种工具函数方法既能赋予 Claude 强大的能力，又能保持代码的组织性和可维护性。

# 工具模式

编写完工具函数后，下一步是创建一个 JSON schema，告诉 Claude 你的函数需要哪些参数以及如何使用它。这个 schema 相当于一份文档，Claude 通过阅读它来了解何时以及如何调用你的工具。

## 理解 JSON Schema

JSON Schema 并非专为 AI 或工具调用而设计——它是一种广泛使用的数据验证规范，已经存在多年。AI 社区之所以采用它，是因为它能够方便地描述函数参数并验证数据。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623699%2F06_-_004_-_Tool_Schemas_01.1748623699583.png)

完整的工具规范包含三个主要部分：

* **name** - 为工具起一个清晰、描述性的名称（如"get_weather"）
* **description** - 工具的功能、使用场景以及返回内容
* **input_schema** - 描述函数参数的实际 JSON schema

## 编写有效的描述

你的工具描述对于帮助 Claude 理解何时使用你的函数至关重要。最佳实践包括：

* 用 3-4 句话说明该工具的功能
* 描述 Claude 应在何时使用它
* 解释它会返回什么类型的数据
* 为每个参数提供详细描述

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623701%2F06_-_004_-_Tool_Schemas_02.1748623700746.png)

## 生成模式的简便方法

你可以使用 Claude 本身来生成 JSON schema，而无需从头编写。具体流程如下：

1. 复制你的工具函数代码
2. 前往 Claude，让它编写一个用于工具调用的 JSON schema
3. 将 Anthropic 关于工具使用的文档作为上下文一并提供
4. 让 Claude 按照最佳实践生成格式规范的 schema

提示词可以这样写："为这个函数编写一个有效的 JSON schema 规范，用于工具调用。请遵循附件文档中列出的最佳实践。"

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623702%2F06_-_004_-_Tool_Schemas_13.1748623701923.png)

## 在代码中实现 Schema

一旦 Claude 生成了你的 schema，将其复制到你的代码文件中。以下是一个推荐的命名模式：

```
def get_current_datetime(date_format="%Y-%m-%d %H:%M:%S"):
    if not date_format:
        raise ValueError("date_format cannot be empty")
    return datetime.now().strftime(date_format)

get_current_datetime_schema = {
    "name": "get_current_datetime",
    "description": "根据指定格式返回当前的日期和时间",
    "input_schema": {
        "type": "object",
        "properties": {
            "date_format": {
                "type": "string",
                "description": "指定返回日期时间格式的字符串。使用 Python 的 strftime 格式代码。",
                "default": "%Y-%m-%d %H:%M:%S"
            }
        },
        "required": []
    }
}
```

使用 `function_name` 后跟 `function_name_schema` 的模式来保持架构的组织性，便于与对应的函数进行匹配。

## 添加类型安全

为了更好地进行类型检查，请从 Anthropic 库中导入并使用 `ToolParam` 类型：

```
from anthropic.types import ToolParam

get_current_datetime_schema = ToolParam({
    "name": "get_current_datetime",
    "description": "返回按照指定格式格式化的当前日期和时间",
    # ... rest of schema
})
```

虽然这对于功能实现并非严格必要，但当你将该模式与 Claude 的 API 配合使用时，它可以防止类型错误，并使你的代码更加健壮。

# 处理消息块

在使用 Claude 的工具功能时，你会遇到一种与之前简单文本响应不同的新响应结构。Claude 现在不再只返回单个文本块，而是可以返回包含文本和工具使用信息的多块消息。

## 进行启用工具的 API 调用

要让 Claude 能够使用工具，你需要在 API 调用中包含 `tools` 参数。以下是请求的结构方式：

```
messages = []
messages.append({
    "role": "user",
    "content": "当前的确切时间是什么，格式为 HH:MM:SS？"
})

response = client.messages.create(
    model=model,
    max_tokens=1000,
    messages=messages,
    tools=[get_current_datetime_schema],
)
```

`tools` 参数接收一个 JSON 模式列表，用于描述 Claude 可以调用的可用函数。

## 理解多块消息

当 Claude 决定使用工具时，它会返回一条包含多个内容块的助手消息。这与你之前处理的简单纯文本响应有很大不同。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623695%2F06_-_005_-_Handling_Message_Blocks_07.1748623695372.png)

多块消息通常包含：

* **文本块** - 人类可读的文本，解释 Claude 正在做什么（例如"我可以帮你查询当前时间。让我为你查找这个信息"）
* **工具使用块** - 关于调用哪个工具以及使用哪些参数的代码指令

ToolUse 块包含：

* 用于追踪工具调用的 ID
* 要调用的函数名称（如 "get_current_datetime"）
* 以字典格式传入的输入参数
* 类型标识 "tool_use"

## 使用多块消息管理对话历史

请记住，Claude 不会存储对话历史——你需要手动管理它。在处理工具响应时，你必须保留完整的内容结构，包括所有块。

以下是如何正确将多块助手消息追加到对话历史记录中的方法：

```
messages.append({
    "role": "assistant",
    "content": response.content
})
```

这样可以同时保留文本块和工具使用块，这对于在后续 API 调用中维护对话上下文至关重要。

## 完整的工具使用流程

![截屏2026-02-11 17.47.16.png](https://fcdn.fxjia.shop/base-service-webserver/78502005/assets/img/2026/02/11/d84af5e3-5045-4cd8-ab94-324aa354a080__556KB____size1856x1028.png)

工具使用流程遵循以下模式：

1. 将用户消息连同工具模式发送给 Claude
2. 接收包含文本块和工具使用块的助手消息
3. 提取工具信息并执行实际函数
4. 将工具结果连同完整对话历史发送回 Claude
5. 接收 Claude 的最终响应

每个步骤都需要仔细处理消息结构，以确保 Claude 拥有提供准确响应所需的完整上下文。

## 更新辅助函数

如果你一直在使用 `add_user_message()` 和 `add_assistant_message()` 等辅助函数，你需要更新它们以处理多块内容。当前版本可能只支持单个文本块，但现在它们需要适应包含工具使用块的更复杂内容结构。

这种多块消息处理对于构建健壮的应用程序至关重要，它能够无缝集成 Claude 的工具功能，同时保持正确的对话流程。

# 发送工具结果

在 Claude 请求工具调用后，你需要执行函数并将结果返回。这样就完成了工具使用的工作流程，为 Claude 提供了它所请求的信息。

## 运行工具函数

当 Claude 返回一个工具使用块时，你需要提取输入参数并调用你的函数。以下是访问工具参数的方法：

```
response.content[1].input
```

这会返回一个包含 Claude 想要传递给函数的参数的字典。由于你的函数需要关键字参数而不是字典，因此需要使用 Python 的解包语法：

```
get_current_datetime(**response.content[1].input)
```

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623700%2F06_-_006_-_Sending_Tool_Results_03.1748623700363.png)## 工具结果块

运行工具函数后，你需要使用工具结果块将结果发送回 Claude。该块放置在用户消息内部，用于告知 Claude 执行工具时发生了什么。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623701%2F06_-_006_-_Sending_Tool_Results_05.1748623701131.png)工具结果块具有几个重要属性：

* **tool_use_id** - 必须与此 ToolResult 对应的 ToolUse 块的 id 相匹配
* **content** - 运行工具后的输出，序列化为字符串
* **is_error** - 如果发生错误则为 True

## 处理多个工具调用

Claude 可以在单次响应中请求多个工具调用。例如，如果用户问"10 + 10 等于多少，30 + 30 等于多少？"，Claude 可能会返回两个独立的 ToolUse 块。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623702%2F06_-_006_-_Sending_Tool_Results_07.1748623702036.png)每个工具调用都有一个唯一的 ID，在返回结果时必须匹配这些 ID。这确保了 Claude 能够知道哪个结果对应哪个请求，即使结果的返回顺序不同。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623703%2F06_-_006_-_Sending_Tool_Results_08.1748623703510.png)## 构建后续请求

您对 Claude 的后续请求必须包含完整的对话历史以及新的工具结果。以下是结构：

```
messages.append({
    "role": "user",
    "content": [{
        "type": "tool_result",
        "tool_use_id": response.content[1].id,
        "content": "15:04:22",
        "is_error": False
    }]
})
```

完整的消息历史现在包含：

* 原始用户消息
* 包含工具使用块的助手消息
* 包含工具结果块的用户消息

## 发起最终请求

发送后续请求时，即使你不期望 Claude 再次调用工具，仍然必须包含工具 schema。Claude 需要该 schema 来理解对话历史中的工具引用。

```
client.messages.create(
    model=model,
    max_tokens=1000,
    messages=messages,
    tools=[get_current_datetime_schema]
)
```

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623704%2F06_-_006_-_Sending_Tool_Results_04.1748623704156.png)Claude 随后会返回一条最终消息，将工具结果整合为面向用户的自然回复。至此，工具使用工作流已全部完成——您已成功让 Claude 通过自定义函数访问实时信息。

# 多轮工具对话

在构建包含多个工具的应用程序时，你需要处理 Claude 可能需要按顺序调用多个工具来回答单个用户问题的场景。例如，如果用户问"从今天起 103 天后是哪一天？"，Claude 需要先获取当前日期，然后在此基础上加上 103 天。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623702%2F06_-_007_-_Multi-Turn_Conversations_with_Tools_02.1748623702753.png)这会创建一个多轮对话模式，Claude 在提供最终答案之前会发出多个工具请求。您的应用程序需要自动处理这种情况。

## 多轮工具模式

以下是 Claude 需要使用多个工具时的幕后工作流程：

1. 用户问："从今天起103天后是哪一天？"
2. Claude 使用工具调用块进行响应，请求 `get_current_datetime`
3. 你的服务器调用该函数并返回结果
4. Claude 意识到需要更多信息，于是请求调用 `add_duration_to_datetime`
5. 你的服务器调用该函数并返回结果
6. Claude 现在有了足够的信息来提供最终答案

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623703%2F06_-_007_-_Multi-Turn_Conversations_with_Tools_03.1748623703654.png)

## 构建对话循环

要处理这种模式，你需要一个对话循环，持续运行直到 Claude 停止请求工具：

```
def run_conversation(messages):
    while True:
        response = chat(messages)
  
        add_user_message(messages, response)
```

## 重构辅助函数

在实现对话循环之前，你需要更新辅助函数以正确处理多个消息块。

### 更新消息处理器

你的 `add_user_message` 和 `add_assistant_message` 函数目前假设处理的始终是纯文本。请更新它们以处理完整的消息对象：

```
from anthropic.types import Message

def add_user_message(messages, message):
    user_message = {
        "role": "user",
        "content": message.content if isinstance(message, Message) else message
    }
    messages.append(user_message)
```

这样你就可以传入字符串、块列表或完整的消息对象。

### 更新聊天函数

修改你的聊天函数，使其接受工具列表并返回完整消息，而不仅仅是文本：

```
def chat(messages, system=None, temperature=1.0, stop_sequences=[], tools=None):
    params = {
        "model": model,
        "max_tokens": 1000,
        "messages": messages,
        "temperature": temperature,
        "stop_sequences": stop_sequences,
    }
  
    if tools:
        params["tools"] = tools
  
    if system:
        params["system"] = system
  
    message = client.messages.create(**params)
    return message
```

### 从消息中提取文本

由于你现在返回的是完整的消息对象，需要创建一个辅助函数来在需要时提取文本：

```
def text_from_message(message):
    return "\n".join(
        [block.text for block in message.content if block.type == "text"]
    )
```

这个函数会查找消息中的所有文本块并将它们拼接在一起，当你需要向用户展示最终响应时非常有用。

# 实现多轮对话

构建带有工具的对话系统需要实现一个循环，持续调用 Claude 直到它停止请求使用工具。当 Claude 不再请求工具时，这表明它已准备好向用户提供最终响应。

## 检测工具请求

判断 Claude 是否想要使用工具的关键在于响应消息中的 `stop_reason` 字段。当 Claude 决定需要调用工具时，该字段会被设置为 `"tool_use"`。这为我们提供了一种简洁的方式来检查是否需要继续对话循环：

```
if response.stop_reason != "tool_use":
    break  # Claude is done, no more tools needed
```

## 对话循环

主对话函数遵循一个简单的模式：

```
def run_conversation(messages):
    while True:
        response = chat(messages, tools=[get_current_datetime_schema])
        add_assistant_message(messages, response)
        print(text_from_message(response))
  
        if response.stop_reason != "tool_use":
            break
  
        tool_results = run_tools(response)
        add_user_message(messages, tool_results)
  
    return messages
```

这个循环会持续进行，直到 Claude 给出最终答案而不再请求任何工具。

```
        "type": "tool_result",
```

## 可扩展的工具路由

为了支持多个工具，需创建一个路由函数，将工具名称映射到其对应的实现：

```
def run_tool(tool_name, tool_input):
    if tool_name == "get_current_datetime":
        return get_current_datetime(**tool_input)
    elif tool_name == "another_tool":
        return another_tool(**tool_input)
    # Add more tools as needed
```

这种方法使得添加新工具变得简单，无需修改核心对话逻辑。

## 完整工作流程

完整的多轮对话工作流程如下：

* 将用户消息连同可用工具发送给 Claude
* Claude 以文本和/或工具请求进行响应
* 执行所有请求的工具并创建结果块
* 将工具结果作为用户消息发送回去
* 重复上述步骤，直到 Claude 提供最终答案

这创造了一种无缝体验，Claude 可以在多轮对话中使用多个工具来完整回答复杂的用户请求。对话历史记录保留了完整的上下文，使 Claude 能够基于之前的工具结果提供全面的回复。
