# 批处理工具

在使用 Claude 的工具调用功能时，你可能会注意到 Claude 可以在单个助手消息中包含多个工具使用块。这使得 Claude 能够并行运行多个工具，而不是为每个工具分别发起请求。然而，在实践中让 Claude 始终如一地做到这一点可能具有挑战性。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623764%2F06_-_010_-_The_Batch_Tool_00.1748623764137.png)例如，如果你要求 Claude 为同一日期设置两个提醒，理想情况下它会发回一个包含两个工具使用块的响应。但通常，Claude 会先进行第一次工具调用，等待你的响应，然后在单独的往返中进行第二次工具调用。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623765%2F06_-_010_-_The_Batch_Tool_04.1748623765474.png)这会造成不必要的来回沟通，而这些操作本可以同时执行。解决方案是实现一个批处理工具，鼓励 Claude 一次请求多个工具调用。

## 批处理工具的工作原理

批量工具是一个额外的工具，它接受对其他工具的调用列表。Claude 不是直接调用多个工具，而是调用批量工具并提供描述它想要运行哪些工具的参数。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623767%2F06_-_010_-_The_Batch_Tool_05.1748623766803.png)批量工具接收一个调用列表，其中每个调用指定：

* 要调用的工具名称
* 传递给该工具的参数

当你收到批量工具调用时，你的代码会遍历调用列表并执行每个请求的工具调用。

## 实现批量工具架构

首先，你需要定义批处理工具的架构。结构如下：

```
batch_tool_schema = {
    "name": "batch_tool",
    "description": "Invoke multiple other tool calls simultaneously",
    "input_schema": {
        "type": "object",
        "properties": {
            "invocations": {
                "type": "array",
                "description": "The tool calls to invoke",
                "items": {
                    "type": "object",
                    "properties": {
                        "name": {
                            "type": "string",
                            "description": "The name of the tool to invoke"
                        },
                        "arguments": {
                            "type": "object",
                            "description": "The arguments to pass to the tool"
                        }
                    }
                }
            }
        }
    }
}
```

## 实现批处理函数

接下来，你需要创建处理批量工具调用的函数：

```
def run_batch(invocations=[]):
    batch_output = []
  
    for invocation in invocations:
        name = invocation["name"]
        args = json.loads(invocation["arguments"])
  
        tool_output = run_tool(name, args)
  
        batch_output.append({
            "tool_name": name,
            "output": tool_output
        })
  
    return batch_output
```

这个函数：

1. 遍历列表中的每个调用
2. 提取工具名称和参数
3. 使用你现有的 `run_tool` 函数调用相应的工具
4. 将所有结果收集到批量输出列表中

你还需要更新主工具路由函数以处理批量工具：

```
def run_tool(tool_name, tool_input):
    if tool_name == "get_current_datetime":
        return get_current_datetime(**tool_input)
    elif tool_name == "add_duration_to_datetime":
        return add_duration_to_datetime(**tool_input)
    elif tool_name == "set_reminder":
        return set_reminder(**tool_input)
    elif tool_name == "batch_tool":
        return run_batch(**tool_input)
```

## 结果

实现后，当 Claude 识别到可以并行执行多个操作时，它将更倾向于使用批量工具。Claude 不会发起单独的请求，而是会调用批量工具并传入它想要执行的所有操作列表。

批量工具本质上是一种变通方法，为 Claude 提供了更高层次的并行工具执行抽象。虽然这种方法看起来可能有些不寻常，但它是一种有效的技术，可以鼓励 Claude 将相关操作批量处理，减少 API 往返次数并提高整体效率。

# 用于结构化数据的工具

当你需要从 Claude 获取结构化数据时，有两种主要方法：使用消息预填充和停止序列的提示词技术，或使用工具的更稳健方法。虽然基于提示词的方法设置更简单，但工具能提供更可靠的输出，代价是增加了复杂性。

## 用于结构化数据的工具

基于工具的方法通过创建 JSON schema 来定义你想要提取的数据的确切结构。你不是寄希望于 Claude 正确格式化其响应，而是本质上给 Claude 提供一个函数来调用，该函数带有与你期望的输出结构匹配的特定参数。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623754%2F06_-_011_-_Tools_for_Structured_Data_03.1748623754770.png)流程如下：

* 编写一个描述你要查找的数据结构的模式
* 使用 `tool_choice` 参数强制 Claude 使用工具
* 从工具使用响应中提取结构化数据
* 无需提供后续响应——一旦获得数据就完成了

例如，如果你想从财务报表中提取余额和关键见解，你的 schema 会分别将它们定义为整数和字符串数组。

## 控制工具使用

这项技术的关键部分是确保 Claude 实际调用你的工具。你可以使用 `tool_choice` 参数来控制此行为：

![img](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623755%2F06_-_011_-_Tools_for_Structured_Data_09.1748623755582.png)

* `{"type": "auto"}` - 模型决定是否需要使用工具（默认）

* `{"type": "any"}` - 模型必须使用工具，但可以选择使用哪一个

* `{"type": "tool", "name": "TOOL_NAME"}` - 模型必须使用指定的工具

对于结构化数据提取，你通常需要第三个选项来保证 Claude 调用你的特定 schema 工具。

## 实现示例

假设你想从文章中提取标题、作者和关键见解。首先，你需要创建一个工具 schema：

```
article_summary_schema = {
    "name": "article_summary",
    "description": "Extracts structured data from articles",
    "input_schema": {
        "type": "object",
        "properties": {
            "title": {"type": "string"},
            "author": {"type": "string"}, 
            "key_insights": {
                "type": "array",
                "items": {"type": "string"}
            }
        }
    }
}
```

然后你使用工具调用 Claude 并强制其使用：

```
response = chat(
    messages,
    tools=[article_summary_schema],
    tool_choice={"type": "tool", "name": "article_summary"}
)
```

响应将包含一个工具使用块，其中 `input` 字段包含你的结构化数据。你可以直接访问它：

```
structured_data = response.content[0].input
```

## 何时使用每种方法

当你需要快速简单的解决方案时，选择基于提示词的结构化输出。当你需要保证可靠性并能处理额外的设置复杂性时，使用工具。根据你的具体用例和需求，这两种技术都很有价值。


# 细粒度工具调用

当你在 Claude 中结合工具使用和流式传输时，AI 在生成工具参数时会实时更新。这创造了更具响应性的用户体验，但关于其幕后工作原理，有一些重要细节需要理解。

## 基本工具流式传输

启用流式传输后，Claude 在处理您的请求时会发送不同类型的事件。您已经熟悉用于常规文本生成的 `ContentBlockDelta` 等事件。对于工具使用，您还需要处理一种名为 `InputJsonEvent` 的新事件类型。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1752775508%2F06_-_011.1_-_Fine_Grained_Tool_Calling_01.1752775507859.png)每个 `InputJsonEvent` 包含两个关键属性：

* **partial_json** - 表示工具参数一部分的 JSON 片段
* **snapshot** - 从目前为止接收到的所有片段累积构建的完整 JSON

以下是你在流式处理管道中处理这些事件的方式：

```
for chunk in stream:
    if chunk.type == "input_json":
        # Process the partial JSON chunk
        print(chunk.partial_json)
        # Or use the complete snapshot so far
        current_args = chunk.snapshot
```

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1752775508%2F06_-_011.1_-_Fine_Grained_Tool_Calling_02.1748623754770.png)## JSON 验证的工作原理

有趣的地方来了。Anthropic API 不会在 Claude 生成时立即发送每个数据块。相反，它会先缓冲数据块并进行验证。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1752775509%2F06_-_011.1_-_Fine_Grained_Tool_Calling_08.1752775509598.png)API 在发送任何内容之前会等待完整的顶级键值对。例如，如果你的工具期望这种结构：

```
{
  "abstract": "This paper presents a novel...",
  "meta": {
    "word_count": 847,
    "review": "This paper introduces QuanNet..."
  }
}
```

API 将：

1. 等待直到整个 `abstract` 值完成
2. 根据你的 schema 验证该键值对
3. 一次性发送 `abstract` 的所有缓冲块
4. 对 `meta` 对象重复此过程

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1752775510%2F06_-_011.1_-_Fine_Grained_Tool_Calling_10.1752775510417.png)这种验证过程解释了为什么即使启用了流式传输，你也会看到延迟后突然出现文本。数据块会被保留，直到准备好一个完整、有效的顶级键值对。

# 细粒度工具调用

如果你需要更快、更细粒度的流式传输——也许是为了向用户显示即时更新或快速开始处理部分结果——你可以启用细粒度工具调用。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1752775512%2F06_-_011.1_-_Fine_Grained_Tool_Calling_13.1752775512126.png)细粒度工具调用主要做一件事：它在 API 端禁用 JSON 验证。这意味着：

* 你可以在 Claude 生成时立即获得数据块
* 顶级键之间没有缓冲延迟
* 更传统的流式行为
* **关键：** JSON 验证被禁用——你的代码必须处理无效的 JSON

通过在 API 调用中添加 `fine_grained=True` 来启用它：

```
run_conversation(
    messages, 
    tools=[save_article_schema], 
    fine_grained=True
)
```

使用细粒度工具调用，你可能在流中更早地收到 `word_count` 值，而无需等待整个 `meta` 对象完成。

## 处理无效 JSON

使用细粒度工具调用时，Claude 可能会生成无效的 JSON，如 `"word_count": undefined` 而不是正确的数字。你的应用程序需要优雅地处理这些情况：

```
try:
    parsed_args = json.loads(chunk.snapshot)
except json.JSONDecodeError:
    # Handle invalid JSON appropriately
    print("Received invalid JSON, continuing...")
```

如果没有细粒度工具调用，API 的验证会捕获此错误，并可能将有问题的值包装在字符串中，这可能与你的预期 schema 不匹配。

## 何时使用细粒度工具调用

在以下情况下考虑启用细粒度工具调用：

* 你需要向用户显示工具参数生成的实时进度
* 你想尽快开始处理部分工具结果
* 缓冲延迟对你的用户体验产生负面影响
* 你能够实现健壮的 JSON 错误处理

对于大多数应用程序，带验证的默认行为是完全足够的。但当你需要额外的响应性时，细粒度工具调用为你提供了控制，让你能够以 Claude 生成的速度获得数据块。

# 文本编辑工具

Claude 带有一个内置工具，你不需要从头创建：文本编辑器工具。此工具使 Claude 能够像在标准文本编辑器中一样处理文件和目录。

## 文本编辑器工具的功能

文本编辑器工具为 Claude 提供了一套全面的文件操作功能：

* 查看文件或目录内容
* 查看文件中特定范围的行
* 替换文件中的文本
* 创建新文件
* 在文件的特定行插入文本
* 撤销最近对文件的编辑

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623830%2F06_-_012_-_The_Text_Edit_Tool_00.1748623830120.png)

这大大扩展了 Claude 的能力，本质上使其能够立即充当软件工程师。

## 理解实现要求

这里有点令人困惑：虽然工具 schema 内置在 Claude 中，但你仍然需要提供实际的实现。这样想——Claude 知道如何请求文件操作，但你需要编写实际执行这些操作的代码。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623831%2F06_-_012_-_The_Text_Edit_Tool_04.1748623831083.png)

当你使用其他工具时，你需要编写 JSON schema 和函数实现。对于文本编辑器工具，Claude 提供 schema 知识，但你必须编写函数来处理 Claude 创建文件、读取目录、替换文本等请求。

## Schema 版本

虽然主 schema 内置在 Claude 中，但在发出请求时你确实需要包含一个小的 schema 存根。确切的 schema 取决于你使用的 Claude 模型：

```
def get_text_edit_schema(model):
    if model.startswith("claude-3-7-sonnet"):
        return {
            "type": "text_editor_20250124",
            "name": "str_replace_editor",
        }
    elif model.startswith("claude-3-5-sonnet"):
        return {
            "type": "text_editor_20241022", 
            "name": "str_replace_editor",
        }
```

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623832%2F06_-_012_-_The_Text_Edit_Tool_12.1748623832064.png)

Claude 看到这个小的 schema 并自动在幕后将其扩展为完整的文本编辑器工具规范。

## 实际示例

让我们看看文本编辑器工具的实际应用。当你要求 Claude 处理文件时，它将使用该工具根据需要读取、修改和创建文件。

例如，如果你要求 Claude "打开 ./main.py 文件并总结其内容"，Claude 将：

1. 使用文本编辑器工具查看文件
2. 读取内容
3. 为你提供摘要

你可以通过要求 Claude 修改文件来进一步扩展。例如："打开 ./main.py 文件并编写一个计算 pi 到第 5 位数字的函数。然后创建一个 ./test.py 文件来测试你的实现。"

Claude 将：

1. 查看现有的 main.py 文件
2. 用包含 pi 计算函数的新实现替换其内容
3. 创建一个带有适当单元测试的新 test.py 文件

## 为什么使用文本编辑器工具？

当现代代码编辑器已经内置 AI 助手时，你可能想知道为什么这个工具存在。文本编辑器工具在以下场景中变得有价值：

* 你正在构建需要以编程方式编辑文件的应用程序
* 你在无法访问功能齐全的代码编辑器的环境中工作
* 你想将文件编辑功能直接集成到你的 Claude 驱动的应用程序中

本质上，文本编辑器工具让你在自己的应用程序中复制许多功能强大的 AI 驱动代码编辑器的功能，让你能够精细控制 Claude 如何与你的文件系统交互。

# 网页搜索工具

Claude 包含一个内置的网络搜索工具，可以让它在互联网上搜索当前或专业信息来回答用户问题。与其他需要你提供实现的工具不同，Claude 会自动处理整个搜索过程——你只需要提供一个简单的架构来启用它。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623824%2F06_-_013_-_The_Web_Search_Tool_00.1748623823785.png)

## 设置网络搜索工具

要使用网络搜索工具，你需要创建一个包含以下必填字段的架构对象：

```
web_search_schema = {
    "type": "web_search_20250305",
    "name": "web_search", 
    "max_uses": 5
}
```

`max_uses` 字段限制了 Claude 可以执行的搜索次数。Claude 可能会根据初始结果进行后续搜索，因此这可以防止过多的 API 调用。单次搜索会返回多个结果，但 Claude 可能会决定需要进行额外的搜索。

## 响应的工作原理

当 Claude 使用网络搜索工具时，响应包含几种类型的块：

* **文本块** - Claude 对其操作的解释
* **ServerToolUseBlock** - 显示 Claude 使用的确切搜索查询
* **WebSearchToolResultBlock** - 包含搜索结果
* **WebSearchResultBlock** - 包含标题和 URL 的单个搜索结果
* **引用块** - 支持 Claude 陈述的文本

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623825%2F06_-_013_-_The_Web_Search_Tool_07.1748623824808.png)

响应结构让你能够准确看到 Claude 搜索了什么以及找到了哪些来源。引用包括 Claude 用来支持其答案的具体文本，以及来源 URL。

## 限制搜索域

你可以使用 `allowed_domains` 字段将搜索限制在特定域名。当你需要可靠的权威来源时，这特别有用：

```
web_search_schema = {
    "type": "web_search_20250305",
    "name": "web_search",
    "max_uses": 5,
    "allowed_domains": ["nih.gov"]
}
```

例如，在询问医疗或运动建议时，限制到 PubMed (nih.gov) 等域名可以确保你获得基于证据的信息，而不是随机的博客内容。

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623825%2F06_-_013_-_The_Web_Search_Tool_13.1748623825691.png)

## 渲染搜索结果

响应中的不同块类型是为特定的 UI 渲染设计的：

* 将文本块渲染为常规内容
* 在顶部以来源列表的形式显示网络搜索结果
* 在文本中内联显示引用，包括来源域名、页面标题、URL 和引用文本

![](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623826%2F06_-_013_-_The_Web_Search_Tool_17.1748623826456.png)

这种结构帮助用户理解 Claude 如何得出答案，并提供关于所使用来源的透明度。引用格式清楚地表明哪些具体信息来自哪些来源，从而建立对 AI 回答的信任。

## 实际应用

网络搜索工具最适合用于：

* 当前事件和最新动态
* Claude 训练数据中没有的专业信息
* 事实核查和查找权威来源
* 需要最新信息的研究任务

只需在进行 API 调用时将该模式包含在工具数组中，Claude 就会自动决定何时进行网络搜索以帮助回答用户的问题。
